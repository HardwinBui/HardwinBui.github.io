Fate Escape

(April 2021)

Roles: Sole Programmer, Sole Game Designer
Download: https://hardwin.itch.io/fate-escape

This a small endless action game I made for a game jam. The core gameplay idea was to often have the player quickly make series of choices in a short time frame to get out of situations where they feel tense. Some things I programmed include:
- Using rigidbodies and colliders to detect collision
- Designing an abstract data structure for the player and enemies
- Repeatedly instantiating random enemies after a set interval of time

The main lesson I learned was to make sure I'm using Unity's provided components rather than re-implementing them myself.

Slime Escape Mobile

(January 2021 - August 2021)

Roles: Sole Programmer, Game Designer
Desktop Download: https://iamcatbone.itch.io/slime-escape
Android Download: https://play.google.com/store/apps/details?id=com.SlimeEscape.SlimeEscape

A desktop puzzle game that was later revised and expanded for a mobile release. The gameplay consists of using power-ups in the correct order to traverse towards the goal of the level. Some things I worked on include:
- Setting up responsive UI and camera bounds
- Implementing mobile video and banner ads that appear in game
- Making prefabs of every type of object to allow for rapid level production
- Gameplay mechanics for each power-up, including tossing rocks at different angles

Glitch Escape

(January 2020 - June 2020)

Roles: UI Programmer
Download: https://glitchescape.itch.io/glitch-escape

A 3D puzzle platformer made with a total of 16 college students. There were 6 other programmers that I'd communicate with on a bi-daily basis to update and connect functions of the game together. I handled the dialouge system and cutscenes of the game, using YarnSpinner to get the text and UI to appear on screen.

This project taught me the value in communicating and being transparent with my teammates. During development, I initially refrained from asking too many questions and struggled to quickly understand code that someone else wrote. I learned that this ultimately creates unnecessary clogs in the development pipeline.

Two Steps Forward

(July 2019 - June 2020)

Roles: Sole Developer
Download: https://hardwin.itch.io/project-ii

This is a small 2D combat-based action game centered around having to swap between two characters. It consists of one level that the player has to traverse through and a boss fight. Some things I worked on include:
- Designing an abstract data structure for the player and enemies
- Organizing scripts so that almost everything is updated through one script
- Creating a boss AI that would pick random sets of actions and act based upon it
